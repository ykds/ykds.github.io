Hexo+Github搭建个人博客

发表于 2017-10-27 | 分类于 other
字数统计: 872 | 阅读时长 ≈ 3

今天用了半天时间，跟着网上的教程搭建了自己的个人博客，由于是第一次搭建，所以花费时间也比较长，也还不是很完善。刚建好的博客，就总结一下搭建的过程来作为我的第一篇文章，虽然这类文章在网上一大堆，但我也决定自己动手来写一篇总结一下搭建过程中遇到的问题，也方便以后搭建时查阅。

1. 安装Node.js和Git

Node.js下载地址

Git下载地址

下载下来后直接安装即可。

2.安装Hexo

进入到Hexo官网，滑到底部，在cmd或git bash下执行那5行命令，为了方便，在下面贴出这5行命令(以官网为准)。

$ npm install hexo-cli -g

$ hexo init blog

$ cd blog

$ npm install

$ hexo server

如若没有出错，则此时可以通过localhost:4000访问你的博客了

3. 将博客部署到Github上

  1. 首先必须到Github上注册一个账号

  2. 新建一个仓库，仓库名必须是yourgithubname.github.io，如我是ykds，则必须是ykds.github.io，写好仓库名直接Creat repository即可

  3. 进入到我们第2步初始化的blog目录，编辑_config.yml文件，拉到文件末尾，作如下修改

    deploy:

    ​ type: git

    ​ repository: https://github.com/yourname/yourname.github.io.git

    ​ branch: master

    注意：在”:”后面有一个空格，repository改为刚才创建的仓库地址

    ​

  4. 安装hexo-deployer-git自动部署发布工具

    npm instal lhexo-deployer-git –save

    ​

  5. 部署到Github

    hexo clean && hexo g && hexo d

    部署过程会弹出Github账号的验证窗口，命令执行完后显示INFO Deploy done: git则说明部署成功，现在可以通过yourname.github.io访问你的博客了。

    ​

4. 将博客绑定到域名(可选)

#####     购买域名

​ 在阿里云网站购买域名(也可以在其它网站购买，因为我是在阿里云购买，所以以此为例)，阿里云域名注册，购买后可以在控制台-域名中查看到你的购买的域名。

#####     域名解析

​ 在控制台-域名中选择你要解析的域名，点击后面的解析

​

​ 然后点击添加解析，进行如下修改，记录值对应的修改为你的博客访问地址

​ 添加完后勾选这个记录，点击启用即可

​

#####     在github仓库添加GNAME文件

​ 以上操作已经成功将你的域名和你的博客绑定，但此时访问你会发现是404，所以我们要建一个CNAME文件将我们的博客映射到个人域名上。

​ 进入仓库页面，点击Create new file，命名为CNAME（没有扩展名)，把个人域名添加到文件，Commit new file即可。

​

​ 此时已经可以通过域名访问你的博客，但是我们只在远程仓库添加了CNAME文件，当我们在本地再次部署时，会出现错误，所以在我们的blog/source目录下同样创建一个一样的CNAME文件，这时再次部署就没有问题了。

​ 最后可以根据参考里的next主题个性化配置来设置你的博客主题，让你的博客更好看。
