##什么是数据结构？

数据结构就是数据与数据之间存在的关系，就像分子结构一样，每个原子之间都有它们之间的关系。

##数据结构类型：

  1. 线性表
  2. 栈
  3. 队列
  4. 树
  5. 图

##线性表

定义

零个或多个数据元素的有限序列。

线性表的存储结构有两种：

  1. 顺序存储结构 （常见的就是数组）
  2. 链式存储结构 （链表）

####顺序存储结构

#####定义

存放一堆相同类型元素的列表，每个元素一个接一个按顺序排列存储，每个元素都只有一个后继，如：

array = [1, 2, 3, 4] √ （这是一个数组， 不是 Python 的 List，List 是 Python 里实现的一种数据结构）

array = [1, ‘a’, 2.5] x

顺序存储在内存中用一段连续的内存地址存储数据，可以把这个连续的内存地址想像为一排排列过去的房子，每个新入住的人按顺序入住，张三来了，住1号；李四来了，住2号；王五来了，住3号·····。

在存储数据之前，要先在内存申请一定的空间，比如，我现在有10个元素的存储，那么要先在内存申请10个存储空间，一旦确定了申请空间的数量就不能改了，申请了10个就最多只能存10个，想要存更多就只能重新申请了。

还是房子举例：

有10个人，大家感情很好，想做个邻居，于是乎找了块地起10间房子（申请内存），房子建好了，突然有3个人不能来了，于是10间房子就只住前面7间，后面还有3间房子空着，如果有人想要住进去，交钱就能拎包入住。

如果10间都住满了，来了一个朋友，看见环境不错哟，诶，我也想住这！不行，不够地了，这块地只能建10间房子了，想要住一直的话只能重新找块能建11间房子的地了，然后这10个人就跟着一直搬过去（重新申请内存）（这坑爹的，一人要加入，全部都要搬）

顺序结构的插入与删除

顺序结构的插入和删除比较麻烦阿。

先用例子解释一番，还是房子。

房子住满了7间，人家前面几家住得好好的，突然来个人说，诶，你这个风水好，我要住这了！如果这样的话，那从这间房子开始，后面的人都要往后搬。 另外，如果10间房子都住满了，你硬塞进其中一个，人家就要打你了，程序报异常了。

所以，插入的时候，选在中间位置插入的话，那么就要移动很多元素，程序的速度也就慢了，如果企图把元素插入已经满的了数组，那就会报异常了，只能放这么多啦，再多就接受不过来啦。

删除操作和插入差不多，中间删掉一个元素，后面的元素就要往前移一位。

顺序存储结构的优点：

可以快速存取表中任一位置的元素，l = [1, 2, 3, 4]，想访问3，直接 l[2]就行了

缺点：

  1. 插入和删除需要移动大量的元素。
  2. 存储的元素个数变化大时，难以确定存储空间容量。 比如： 我现在有10个元素要存储，但以后可能要增加到15个，20个，也可能减少到8个，5个。

链式存储结构

由于顺序存储结构在插入和删除时需要移动大量元素，导致程序运行速度减慢，这时就要用到链式存储的方式。

链式存储结构就是第一个元素结点，除了记录本身的信息外，还要记录它下一个元素结点的位置，链式存储在内存中不是连续的，只是逻辑上是一个线性的结构。

链式存储结构对应的就是链表，链表的结构就像这样：

存储数据的就叫做数据域，存储指针的就是指针域，指针的意思就是指向下一个元素内存地址的一个东西。A, B, C 就称为结点。

我们把指向第一个结点的指针就称为头指针，可以理解为这个头指针就是第一个结点在内存的地址。

有时，为了操作方便，还会在链表的第一个结点前加一个叫做头结点的结点，这个结点的数据域可以不存储东西，但指针域指向第一个结点（这里第一个结点不是头结点，头结点后面那个才是第一个结点），头结点不是必须的。

单链表的读取：

链表的可以解决插入和删除的麻烦（下面讲解），但是也带来了读取的麻烦，它不像顺序结构那么随心所欲的访问啦，因为链表的数据是分散在内存的，某个元素的哪只有它上一个元素知道。如：

我想找王五，只能先找到李四，要找到李四，先找到张三，只能一个一个找下去。

代码读取像这样，我要找王五！从表头开始找，第一个是张三，不是找你，从张三这我知道了下一个人的位置，继续找，找到了李四，也不是你，又找到下一个，一对比，诶，是你了，王五，读取数据。如果在这个链表中间还有很多个结点，我想找马六，这查找量就变大了，最坏情况，马六根本不在这里。

单链表的插入与删除：

插入、删除就方便多了，因为每个元素是分散在内存的，这条链表上的增删对其它结点没什么影响。

我要在a和b之间插入c，只要把a原本指向b的指针改为指向c，然后c指向再b，就大功告成了。

代码像这样： c.next = a.next, a.next = c，两步操作。 next就是指向下一个元素的指针。a.next 就是b，先把 a.next 赋值给 c.next ，让 c 指向了 b, 再让a.next指向 c 。 如果这两条代码调转，结果就不一样了，a.next = c, 那么 c.next = a.next 的话，c 不就指回自己了吗，链表就断啦。

删除也很简单：

我要删除 b，直接 a.next = b.next， 那 a 不就指向 c 是吗，b 就离开了这条链表啦，如果想删除最后的c，直接让 b 指向 null，那链表就到 b 为止啦。

顺序结构和链式结构的优缺点：

  1. 顺序结构可以快速存取，链式结构只能一个一个去找
  2. 顺序结构插入和删除数据需要移到大量元素，链式结构插入和删除数据只需要改为指向结点
  3. 顺序结构申请空间后就固定了，分小了，不够用，分大了，浪费空间；链式结构只要内存里还有位置就能申请空间，然后把新元素添加到链表就行。

关于链表的只是简单介绍了单链表，还有循环链表和双向链表，以免一次接受过多，先到这里。
